## 一、GitHub About（仓库简介）

填进 **Settings → About → Description**（建议 1–2 句，不超过 350 字符）。

**主版本：**

```
Self-hosted, multi-protocol API client — HTTP/2, WebSocket, MQTT, Socket.IO, gRPC, GraphQL & raw TCP/UDP relay. One Go binary serving a Vue 3 UI, team roles, saved collections, dark mode, 中文/EN.
```

**更短版本（适合做 social card 副标题）：**

```
Open-source Postman alternative that also speaks MQTT, gRPC & TCP/UDP — self-hosted, single binary, bilingual.
```

## 三、README 头部改写（已应用）

README 开头已改写为更抓人的版本，要点：

- 加了一句 tagline：`> Self-hosted, multi-protocol API client — one Go binary, no cloud lock-in.`
- 第一段点明**自托管 + 单二进制**的核心价值：请求、token、API key 不出自有基础设施。
- 第二段突出**多协议壁垒**（MQTT / Socket.IO / gRPC / TCP-UDP），并直接对比「大多数 API 客户端只做 REST & GraphQL」。
- 原 `## Features` 合并为 `## Why Apiforge`，用加粗短句列出 5 个卖点。

当前 README 头部结构：

```
# Apiforge
> Self-hosted, multi-protocol API client — one Go binary, no cloud lock-in.
[License badge] [CI badge]
**Apiforge** is an open-source API client ...
## Why Apiforge
- Self-hosted & single-binary
- Multi-protocol in one tool
- Team-ready
- Bilingual & themed
- Portable storage
Planned: MCP and AI / LLM endpoint debugging.
```

其余内容（Tech stack / Quick start / Configuration / Roadmap / 安全审计）保持不变。
